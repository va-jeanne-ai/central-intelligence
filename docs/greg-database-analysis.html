01The one-paragraph version

Greg's database isn't a CRM export — it's a full sales-and-marketing intelligence platform that has been quietly accumulating exactly the data Central Intelligence was designed to consume. CI's own schema mirrors Greg's domain almost table-for-table, but CI's tables are empty or seed-fed while Greg's are full of real, AI-enriched data.

The headline: Central Intelligence is a well-built engine running on an empty tank. Greg's database is the fuel — 11,561 real leads, 103 scored call transcripts, 280 customer-psychology insights, 2,390 email campaigns, 7,566 webinar engagements, and 18,932 logged sales activities. The work ahead isn't mostly building features; it's wiring the engine to the fuel and then building the handful of net-new things Greg's data makes possible.

02What this database actually is

It began life as a GoHighLevel CRM mirror — but it has grown into the operational brain of Greg's coaching business.

🏢 Two businesses, one tenant root

business_profile holds two brands. Nearly all live data is the first.

  • "The WGR" (business_id=1) — coaching for mid-career real estate agents (ages 28–48, $200–300K, 8–20 yrs experience). The primary org; all reps belong here.
  • "AI Growth Engine" (business_id=2) — a second brand for coaches / consultants / agencies. Minimal data.

Product line: the "Agent Infopreneur" coaching ladder — Jumpstart / Accelerator / Mastery, priced $2K–$15K.

🔌 How data gets in

External sync jobs (not ours) populate it from several sources:

  • GoHighLevel → leads, appointments, calls, conversations
  • EverWebinar → webinar engagements + opt-ins (dominant top-of-funnel)
  • Instagram / Facebook → posts, comment-keyword automation
  • ESP (ActiveCampaign-style) → email campaign performance
  • An AI scoring pipeline → transcripts → scores → analyses → insights
Two clocks to keep in mind. The CRM / webinar / opt-in / email layers go back to 2016–2024. The call-intelligence layer is brand-new — calls, sales_call_*, sales_activities, insights and EOD reports all start around 2026-05-26. The mirror itself was (re)built starting 2026-05-11. So you have years of funnel history but only weeks of deep call intelligence.

08Where Central Intelligence stands — honest maturity

CI is more built than it looks. The spine is real. The soft spots are concentrated in marketing-creative sub-pages, paid ads, and the fact that nearly every data table is waiting to be fed.

Built & real

  • Auth (Supabase JWT)
  • Agent hierarchy: CI → 3 Directors → 11 Specialists
  • Leads / Members / Appointments CRUD
  • Call transcribe + analyzer pipeline
  • RAG embed pipeline (Voyage + pgvector)
  • Integrations framework (encrypted creds)
  • Dashboard, Chat (SSE + WS), Accountability

Built but mock / seed

  • CI chat falls back to mock agent without API key
  • Funnels, Offers, DM — real shell, seed/hardcoded display data
  • Market signals / insights pages — wired but 0 rows
  • Ad generator — MOCK_VARIANTS
  • DM templates & conversations — hardcoded constants

Stub / absent

  • Meta / Google Ads connector — none
  • Sales reps + coaching-strike system — none
  • Closed-sales / revenue tracking — none
  • Webinar + opt-in ingestion — none
  • Org-tier rollups (CEO→Director→Manager→Agent)
  • Multi-tenancy (tenant_id hooks only)
Read this carefully: most of CI's "empty" surfaces aren't unbuilt — they're unfed. The leads page, insights page, market-signals page, content-ideas page, and call-analysis pipeline are all wired to real endpoints that return nothing because their tables are empty. Greg's database is precisely the missing input.

10Data quality & freshness — read before ingesting

Anything we sync inherits these characteristics. Plan mappings around them.

⚠️ Attribution & key hygiene

  • rep_id is the clean key — name strings are dirty (double-spaces, partials). Always join on rep_id.
  • sales_activities.rep_id ~83% null (15,743 of 18,932) — major attribution gap.
  • 30 of 74 sales unattributed to a rep.
  • pipeline_stage 94% null — derive funnel from appointment outcome + call result instead.

⚠️ Coverage & missing content

  • email_campaigns.body_copy null on all 2,390 — metrics only, no creative for RAG.
  • appointments.call_id FK declared but 0 populated — appointments not yet linked to calls.
  • Transcript pipeline: 264 pending — 193 processed, 54 failed, 15 pending. transcript_labeled null on a large share.
  • Test data in calls: "TEST_T1_Alice" / voicemail rows from Greg inflate the recent count — filter on ingest.

⏱️ Two clocks

  • CRM / webinar / opt-in / email: 2016–2024 → present.
  • Call intelligence / activities / insights / EOD: only since ~2026-05-26.
  • Mirror (re)built starting 2026-05-11; counts still growing (calls 109→145 since first look).

🟦 20 empty tables = forward scaffolding

  • Entire broll_* video pipeline (with pgvector).
  • Most voice_* and creator_* (except vision/stories).
  • All DM tables, lead_engagements.
  • meta_ads / meta_ad_performance / meta_creative_tests.
Safety reminder: WGR_DATABASE_URL is the write-capable postgres role. All access goes through wgr_client.py which forces read-only sessions. We never write to Greg's database. A dedicated read-only role has been requested from the client to remove the footgun entirely.
